# パイプライン設計ドキュメント

このドキュメントはパイプライン全体の設計・規約・原則を定義する。
新しいエージェントを追加する際や、既存エージェントの役割を変更する際はここを参照すること。

---

## 目的

複数の Claude Code インスタンスが協調して、テーマに関する調査記事を生成するパイプラインを構成する。
各エージェントは独立した役割を持ち、orchestrator を中継点として疎結合に連携する。

---

## パイプライン構成

```
orchestrator
    │
    ├─ writer     ← サブテーマの記事を執筆
    │     │
    ├─ reviewer   ← 論理整合性・事実確認・ウェブ検索
    │     │
    └─ editor     ← 全サブテーマを統合してサマリーを生成
```

---

## ロール定義

### orchestrator
- テーマを選択してサブテーマに分解する
- 各エージェントへの通知とルーティングを担う
- writer完了 → reviewer、全サブテーマ完了 → editor という順序を制御する
- writerとreviewerは並列実行可（reviewer(n)処理中にwriter(n+1)を走らせてよい）

### writer
- 与えられたサブテーマについて調査し記事を書く
- 執筆上の判断・迷い・情報不足を `_memo.md` に記録する

### reviewer
- writerの記事と `_memo.md` を読んでレビューする
- 論理整合性・網羅性を検証し、不確かな事実はウェブ検索で確認する
- レビュー結果・残課題を `_memo.md` に追記する
- レビュー済み記事を `_reviewed.md` として保存する

### editor
- 全サブテーマの `_reviewed.md` と `_memo.md` を読んでサマリーを生成する
- `_memo.md` の留保・残課題をサマリーの考察に反映する

---

## 通信プロトコル

エージェント間の通信は tmux の `send-keys` によるメッセージパッシング。
完了通知のみをメッセージで送り、思考のコンテキストはファイルで渡す。

### メッセージフォーマット

| 送信者 | フォーマット |
|-------|------------|
| writer | `writer完了: shared/<テーマ名>/<サブテーマ名>.md` |
| reviewer | `reviewer完了: shared/<テーマ名>/<サブテーマ名>_reviewed.md (修正あり\|修正なし)` |
| editor | `editor完了: shared/<テーマ名>/summary.md` |

### tmuxペイン番号

| ペイン | ロール |
|-------|-------|
| agents:0.0 | orchestrator |
| agents:0.1 | writer |
| agents:0.2 | reviewer |
| agents:0.3 | editor |

---

## ファイル命名規約

```
shared/
  <テーマ名>/                          # 実行単位。テーマごとに分離される
    <サブテーマ名>.md                   # writer が生成する記事（原文）
    <サブテーマ名>_memo.md             # writer→reviewer が蓄積する思考メモ
    <サブテーマ名>_reviewed.md         # reviewer が生成するレビュー済み記事
    summary.md                         # editor が生成するサマリー
```

---

## 設計原則

### 疎結合と制御の維持
エージェント同士は直接通信せず、orchestrator を中継点にする。
誰が何を決めたかを常に orchestrator が把握できる状態を保つ。

### 思考コンテキストの受け渡し
成果物（`.md`）だけでなく、思考の経緯（`_memo.md`）を次のエージェントに渡す。
LLMの内部推論はコンテキストウィンドウの中に閉じており、明示的にファイルに書き出さない限り他のエージェントには伝わらない。
tmuxメッセージは揮発性のため、コンテキストはファイルで永続化する。

### run isolation
テーマ名ディレクトリが実行単位となり、複数テーマの成果物が混在しない。

---

## 新しいエージェントを追加する場合

1. `<ロール名>/CLAUDE.md` を作成し、役割・手順・完了通知フォーマットを定義する
2. このドキュメントのロール定義・通信プロトコル・ファイル命名規約を更新する
3. `start.sh` にペインを追加し、tmuxペイン番号を更新する
4. orchestratorの `CLAUDE.md` にルーティングを追加する
